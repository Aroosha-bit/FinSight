## Overview

**FinSight** is a modern personal expense tracking application designed to give users complete control over their finances. Users can log expenses by category, set monthly budgets, view real-time spending trend charts, and receive personalised financial advice generated by Google Gemini AI — all within a clean, responsive interface.

---

## Features

- 🔐 **Authentication** — Secure signup, login, forgot password, and password reset via email link
- 📊 **Real-Time Dashboard** — Monthly spending totals, budget progress bars, and month-over-month comparisons
- 💸 **Expense Management** — Add, edit, delete, filter, and sort expenses across 9 categories
- 📈 **Charts** — Spending Trend line chart (last 6 months) and Category Breakdown donut chart
- 🤖 **Gemini AI Advice** — Three personalised financial tips generated by Gemini `gemini-1.5-flash`, with a smart rule-based fallback when no API key is configured
- 💱 **Multi-Currency Support** — Choose from PKR, USD, and EUR; all figures update across the entire app
- 👤 **Profile Management** — Update name, email, password, and preferred currency at any time
- 🔔 **Notifications** — Per-user notification management
- 📱 **Responsive Design** — Works on desktop and mobile viewports

## Getting Started

### Prerequisites

- **Node.js** v18 or higher
- **npm** v8 or higher
- A running **MongoDB** instance (local: `mongodb://localhost:27017` or MongoDB Atlas)
- *(Optional)* A **Google Gemini API Key** for live AI advice — [Get one here](https://aistudio.google.com/app/apikey)
- *(Optional)* A Gmail **App Password** for password reset emails

---

### Environment Variables

Create a `.env` file inside the `server/` directory:

```env
PORT=5000
MONGO_URI=mongodb://localhost:27017/finsight
JWT_SECRET=your_super_secret_jwt_key_here

# Email settings (for password reset)
EMAIL=your_email@gmail.com
APP_PASSWORD=your_gmail_app_password

# Optional: enables live Gemini AI advice
GEMINI_API_KEY=your_gemini_api_key_here
```

> **Note:** If `GEMINI_API_KEY` is not set, the app will automatically fall back to a local rule-based advice engine and remain fully functional.

---

### Installation

#### 1. Clone the repository

```bash
git clone https://github.com/your-username/ExpenseTrack.git
cd ExpenseTrack
```

#### 2. Install server dependencies

```bash
cd server
npm install
```

#### 3. Install client dependencies

```bash
cd ../client
npm install
```

---

### Running the App

#### Start the backend server (from `server/` directory)

```bash
npm run dev
```

The API will be available at `http://localhost:5000`

#### Start the frontend dev server (from `client/` directory)

```bash
npm run dev
```

The app will open at `http://localhost:5173`

## API Reference

All protected routes require:
```
Authorization: Bearer <jwt_token>
```

### Auth

| Method | Endpoint | Access | Description |
|--------|----------|--------|-------------|
| `POST` | `/api/auth/register` | Public | Register new user |
| `POST` | `/api/auth/login` | Public | Login and receive JWT |
| `POST` | `/api/auth/forgot-password` | Public | Send password reset email |
| `POST` | `/api/auth/reset-password` | Public | Reset password via token |
| `GET` | `/api/auth/profile` | Protected | Get current user profile |
| `PUT` | `/api/auth/profile` | Protected | Update name, email, password, currency |
| `GET` | `/api/auth/advice` | Protected | Get 3 Gemini-powered financial tips |

### Expenses

| Method | Endpoint | Access | Description |
|--------|----------|--------|-------------|
| `POST` | `/api/expense/create-expense` | Protected | Add a new expense |
| `GET` | `/api/expense/allExpense` | Protected | Get all user expenses |
| `GET` | `/api/expense/getExpense/:id` | Protected | Get single expense |
| `PUT` | `/api/expense/updateExpense/:id` | Protected | Update an expense |
| `DELETE` | `/api/expense/deleteExpense/:id` | Protected | Delete an expense |

### Budget

| Method | Endpoint | Access | Description |
|--------|----------|--------|-------------|
| `POST` | `/api/budget/add-budget` | Protected | Create monthly budget |
| `GET` | `/api/budget/allBudgets` | Protected | Get user's budgets |
| `PUT` | `/api/budget/updateBudget/:id` | Protected | Update a budget |

## Gemini AI Advice

When a valid `GEMINI_API_KEY` is provided in the server `.env`, the `/api/auth/advice` endpoint sends a structured prompt to **Gemini 1.5 Flash** containing:

- User's total monthly budget and currency preference
- Current month's total spending
- Per-category spending breakdown

Gemini returns exactly **three personalised financial advice cards** with a relevant emoji icon and concise tip text.

**Fallback behaviour:** If the API key is missing or the Gemini request fails, the server automatically applies a local rule engine — evaluating over-budget categories, transport and food spending thresholds, and savings progress — to generate three equally relevant advice cards without any errors shown to the user.
